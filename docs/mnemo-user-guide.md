# Руководство по работе с мнемосхемами

## Что такое мнемосхема

Мнемосхема — это упрощённое изображение пневматической системы (трубопроводы, клапаны, датчики и т.д.), на котором можно в интерактивном режиме отрабатывать порядок действий оператора. Система позволяет создавать схемы реальных пневмощитков и тренировать специалистов на сценариях — как штатных, так и аварийных.

---

## Типы элементов

### Управляемые (оператор может переключать)

**Клапан (VALVE)** — запорная арматура. В реальной системе это вентили типа АУ341, АУ313 и т.п. Оператор открывает/закрывает их для подачи или перекрытия воздуха в магистрали.

**Насос (PUMP)** — насос или компрессор подачи рабочей среды.

**Переключатель (SWITCH)** — элемент ручного управления (тумблер, кнопка).

**Нагреватель (HEATER)** — подогрев воздуха или иной среды.

**Блокировка (LOCK)** — механизм блокировки/разблокировки участка системы.

**Редуктор (REDUCER)** — редуктор давления. Снижает давление воздуха с высокого на более низкое. Например, АУМО-400 снижает с 230 до 55 ати, АУ311-300 — дальше до рабочего уровня.

### Пассивные (не управляются оператором)

**Датчик давления (SENSOR_PRESSURE)** — манометр. Показывает текущее давление в точке установки. Для датчиков можно задать диапазон (min/max), например 0–300 кг/см² для манометра МВ1-300.

**Датчик температуры (SENSOR_TEMPERATURE)** — термометр. Показывает температуру.

**Предохранительный клапан (SAFETY_VALVE)** — срабатывает автоматически при превышении давления. В реальности это клапаны типа АУМН-600 с настраиваемым порогом 35–65 ати. Оператор его не переключает.

**Фильтр (FILTER)** — фильтр очистки воздуха (например, Д060-060). Стоит на входе магистрали, пропускает поток через себя.

**Обратный клапан (CHECK_VALVE)** — пропускает воздух только в одном направлении. Используется на дренажных линиях для сброса в атмосферу.

**Надпись (LABEL)** — текстовая подпись на схеме, не участвует в логике.

---

## Диапазоны датчиков

У датчиков давления и температуры можно задать собственные min/max значения. Если не заданы — используются значения по умолчанию (0.5–6.0 МПа для давления, 15–85 °C для температуры). Настраивается в редакторе схемы: выделите датчик и укажите нужные значения в панели свойств справа.

## Как пользоваться (для специалиста)

1. Зайти в раздел «Мнемосхемы» → список доступных сценариев.
2. Выбрать сценарий и нажать «Начать».
3. Откроется схема. Вверху — инструкция текущего шага (что нужно сделать).
4. Кликайте по элементам на схеме, чтобы переключить их (открыть/закрыть клапан, включить редуктор и т.д.).
5. Когда состояние элементов совпадёт с ожидаемым — шаг засчитывается, появляется следующий.
6. Пассивные элементы (датчики, фильтры, обратные клапаны, предохранительные клапаны) нельзя переключить — при клике появится информационное сообщение.
7. После прохождения всех шагов — результат (успех/неуспех, время, ошибки).

---

## Как создавать схемы (для начальника)

1. Раздел «Мнемосхемы» → «Схемы» → «Создать схему».
2. Указать название, описание, размеры поля.
3. В редакторе схемы: слева палитра элементов. Перетаскивайте элементы на поле.
4. Для соединения: выберите режим соединения, кликните по элементу-источнику, затем по элементу-приёмнику.
5. Свойства элемента (имя, тип, поворот, диапазон для датчиков) — в панели справа при выделении.
6. Сохранить схему.

Именование элементов — рекомендуется по обозначениям из документации: VP1, VP2 для клапанов, PT1 для датчиков давления, RD1 для редукторов и т.д.

---

## Как создавать сценарии

1. Раздел «Мнемосхемы» → «Сценарии» → «Создать сценарий».
2. Выбрать схему, указать название, описание, лимит времени (минуты).
3. Тип сценария: **Штатный** (NORMAL) или **Аварийный** (FAULT).
4. Для аварийного — указать родительский (штатный) сценарий.
5. Добавить шаги по порядку. Каждый шаг содержит:
   - **Номер шага** — порядок выполнения.
   - **Инструкция** — текст, который увидит оператор.
   - **Ожидаемое состояние** — JSON вида `{"VP1": true, "RD1": true}`. Какие элементы должны быть включены/выключены для завершения шага.
   - **Аварийное событие** (необязательно) — если на этом шаге происходит нештатная ситуация.
   - **Запрещённые действия** (необязательно) — что оператор НЕ должен делать на этом шаге.
   - **Лимит времени шага** (необязательно) — в секундах, для конкретного шага.

### Аварийный сценарий — как устроен

Аварийный сценарий обычно повторяет начало штатного, но на одном из шагов возникает нештатная ситуация. Не нужно делать все шаги аварийными — достаточно 1–2 шагов с аварийным событием, остальные могут быть обычными.

Пример: штатный сценарий — 5 шагов. Аварийный — те же первые 2 шага, на 3-м шаге авария (давление превышено), оператор должен стравить через дренаж, затем продолжить штатную работу.

---

## Типы аварийных событий

Задаются в поле `fault_event` шага в формате JSON.

**ELEMENT_FAILURE — Отказ элемента**
Элемент выходит из строя (клапан заклинило, насос не работает). Если `lockElement: true` — элемент блокируется и оператор не может его переключить. Нужно обходить отказ другими средствами.

Пример: `{"type":"ELEMENT_FAILURE","elementName":"VP5","message":"Клапан VP5 заклинило в закрытом положении!","lockElement":true}`

**PRESSURE_ANOMALY — Аномалия давления**
Давление в точке замера выходит за допустимые пределы. Оператор должен предпринять действия по нормализации (стравить давление, перекрыть подачу и т.д.).

Пример: `{"type":"PRESSURE_ANOMALY","elementName":"PT2","message":"Давление на выходе редуктора превышает 75 ати!","lockElement":false}`

**OVERHEAT — Перегрев**
Температура превышает норму. Актуально для систем с подогревом.

Пример: `{"type":"OVERHEAT","elementName":"TT1","message":"Температура превышает допустимую!","lockElement":false}`

**FALSE_ALARM — Ложное срабатывание**
Сигнализация сработала, но реальной проблемы нет. Оператор должен распознать ложную тревогу и продолжить работу, не предпринимая аварийных действий.

---

## Запрещённые действия

Задаются в поле `forbidden_actions` шага. Это массив JSON — список того, что оператор не должен делать на данном шаге.

Формат:
```json
[
  {
    "elementName": "VP2",
    "action": "on",
    "penalty": "FAIL",
    "message": "Подача воздуха при аномальном давлении запрещена!"
  }
]
```

Поля:
- `elementName` — какой элемент нельзя трогать.
- `action` — какое действие запрещено: `"on"` (включение) или `"off"` (выключение).
- `penalty` — последствия нарушения:
  - **FAIL** — немедленный провал. Сценарий завершается с ошибкой.
  - **WARNING** — предупреждение. Действие выполнится, но ошибка зафиксируется.
  - **TIME_PENALTY** — штраф по времени (минус 30 секунд от общего таймера).
- `message` — текст, который увидит оператор при нарушении.

## Пример: сценарий для пневмощитка 8У016

Штатный сценарий «Подготовка к работе»:
1. Открыть входной вентиль VP1 (подача 230 ати)
2. Включить редуктор RD1 (снижение до 55 ати)
3. Включить редуктор RD2 (снижение до рабочего)
4. Открыть вентиль наддува VP2
5. Открыть вентиль заполнения VP3

Аварийный сценарий «Превышение давления»:
1. Открыть VP1 (как в штатном)
2. Включить RD1 (как в штатном)
3. **АВАРИЯ**: давление на PT2 превышает норму → открыть дренажный вентиль VP4 (запрещено открывать VP2 — штраф FAIL)
4. Давление нормализовалось → включить RD2 и открыть VP2
5. Открыть VP3
