# Взвешенная система оценивания тестов

## Общая модель

В системе применяется взвешенное оценивание, при котором для каждого вопроса рассчитываются:

- `r_i` — коэффициент оценивания ответа на вопрос
- `z_i` — уровень сложности вопроса

Итоговый процент прохождения теста вычисляется по формуле:

`R = (Σ(r_i * z_i) / Σ z_i) * 100`

где:

- `0 <= r_i <= 1`
- `z_i ∈ {1, 2, 3}`
- `R` — итоговый процент результата по тесту

## Уровни сложности

- `z_i = 1` — базовый вопрос
- `z_i = 2` — вопрос средней сложности
- `z_i = 3` — вопрос повышенной сложности

Если для вопроса сложность явно не задана, используется значение `1`.

## Формулы по типам вопросов

### SINGLE_CHOICE

Для вопроса с одним правильным вариантом:

- `r_i = 1`, если выбран правильный ответ
- `r_i = 0`, если выбран неправильный ответ или ответ отсутствует

### MULTIPLE_CHOICE

Для вопроса с несколькими правильными вариантами применяется мягкое частичное оценивание:

`r_i = max(0, correctSelected - wrongSelected) / correctTotal`

где:

- `correctSelected` — количество правильно выбранных вариантов
- `wrongSelected` — количество ошибочно выбранных вариантов
- `correctTotal` — общее число правильных вариантов в вопросе

Если пользователь отметил только неправильные варианты, коэффициент принимается равным `0`.

### MATCHING

Для вопроса на установление соответствия:

`r_i = correctlyMatchedPairs / totalPairs`

где:

- `correctlyMatchedPairs` — число верно составленных пар
- `totalPairs` — общее количество пар в вопросе

### SEQUENCE

Для вопроса на установление правильной последовательности:

`r_i = correctlyPositionedItems / totalItems`

где:

- `correctlyPositionedItems` — количество элементов, стоящих на правильных позициях
- `totalItems` — общее число элементов последовательности

### OPEN_TEXT

Для открытого вопроса применяется автоматическая дихотомическая оценка:

- `r_i = 1`, если ответ совпадает с эталоном без учета регистра и лишних пробелов по краям
- `r_i = 0`, если ответ не совпадает или отсутствует

## Пример расчета

Пусть тест состоит из трех вопросов:

1. Вопрос 1: `r_1 = 1`, `z_1 = 1`
2. Вопрос 2: `r_2 = 0.5`, `z_2 = 2`
3. Вопрос 3: `r_3 = 0`, `z_3 = 3`

Тогда:

- `Σ(r_i * z_i) = 1 * 1 + 0.5 * 2 + 0 * 3 = 2`
- `Σ z_i = 1 + 2 + 3 = 6`
- `R = (2 / 6) * 100 = 33.3%`

## Демонстрация на тесте

Для демонстрации в системе используется тест `Демонстрация взвешенного оценивания`.

Структура теста:

- Вопрос 1 — `SINGLE_CHOICE`, сложность `z = 1`
- Вопрос 2 — `SINGLE_CHOICE`, сложность `z = 1`
- Вопрос 3 — `MULTIPLE_CHOICE`, сложность `z = 2`
- Вопрос 4 — `MATCHING`, сложность `z = 2`
- Вопрос 5 — `SEQUENCE`, сложность `z = 3`

Суммарная сложность теста:

`Σ z_i = 1 + 1 + 2 + 2 + 3 = 9`

### Сценарий 1. Зачёт за счёт сложного вопроса

Ответы:

- Вопрос 1: неверно
- Вопрос 2: верно
- Вопрос 3: выбрать только `Один`
- Вопрос 4: сделать 2 пары из 4 верно
- Вопрос 5: выполнить полностью верно

Коэффициенты:

- Вопрос 1: `r_1 = 0`
- Вопрос 2: `r_2 = 1`
- Вопрос 3: `r_3 = 0.5`
- Вопрос 4: `r_4 = 0.5`
- Вопрос 5: `r_5 = 1`

Расчёт:

`Σ(r_i * z_i) = 0*1 + 1*1 + 0.5*2 + 0.5*2 + 1*3 = 6`

`R = 6 / 9 * 100 = 66.7%`

Итог:

- результат выше проходного балла `60%`
- тест засчитывается, несмотря на ошибку в лёгком вопросе

### Сценарий 2. Тот же набор ответов, но сложный вопрос решён частично

Ответы:

- Вопрос 1: неверно
- Вопрос 2: верно
- Вопрос 3: выбрать только `Один`
- Вопрос 4: сделать 2 пары из 4 верно
- Вопрос 5: ответить `Один, Два, Четыре, Три`

Коэффициенты:

- Вопрос 1: `r_1 = 0`
- Вопрос 2: `r_2 = 1`
- Вопрос 3: `r_3 = 0.5`
- Вопрос 4: `r_4 = 0.5`
- Вопрос 5: `r_5 = 0.5`

Расчёт:

`Σ(r_i * z_i) = 0*1 + 1*1 + 0.5*2 + 0.5*2 + 0.5*3 = 4.5`

`R = 4.5 / 9 * 100 = 50.0%`

Итог:

- результат ниже проходного балла
- тест не засчитывается

Этот сценарий наглядно показывает, что частичный ответ на вопрос сложности `3` даёт вклад `1.5`, а полный ответ на тот же вопрос дал бы вклад `3.0`

### Сценарий 3. Верны только лёгкие вопросы

Ответы:

- Вопрос 1: верно
- Вопрос 2: верно
- Вопрос 3: неверно
- Вопрос 4: неверно
- Вопрос 5: неверно

Расчёт:

`Σ(r_i * z_i) = 1*1 + 1*1 + 0*2 + 0*2 + 0*3 = 2`

`R = 2 / 9 * 100 = 22.2%`

Итог:

- только лёгкие вопросы не позволяют набрать проходной процент

### Сценарий 4. Верен только самый сложный вопрос

Ответы:

- Вопрос 1: неверно
- Вопрос 2: неверно
- Вопрос 3: неверно
- Вопрос 4: неверно
- Вопрос 5: верно полностью

Расчёт:

`Σ(r_i * z_i) = 0*1 + 0*1 + 0*2 + 0*2 + 1*3 = 3`

`R = 3 / 9 * 100 = 33.3%`

Итог:

- один сложный вопрос заметно влияет на результат, но не может в одиночку обеспечить зачёт

## Как демонстрировать в интерфейсе

Для демонстрации работы новой системы оценивания рекомендуется:

1. Выполнить SQL-скрипт `scripts/sql/seed_weighted_demo_test.sql`.
2. Войти в систему под сотрудником, которому назначен демонстрационный тест.
3. Пройти тест по сценарию 1 и зафиксировать итоговый процент `66.7%`.
4. Пройти тест повторно по сценарию 2 и зафиксировать итоговый процент `50.0%`.
5. Сравнить две попытки и показать, что меняется только ответ на вопрос сложности `3`, но итоговый процент меняется существенно.
6. Открыть страницу результата и показать по каждому вопросу:
   - тип вопроса
   - уровень сложности
   - коэффициент ответа `r_i`
   - статус `Полный зачет`, `Частичный зачет` или `Без зачета`
7. Отдельно обратить внимание, что:
   - ошибка в лёгком вопросе не всегда критична
   - частичный ответ на сложный вопрос влияет на итог сильнее, чем частичный ответ на лёгкий
   - итог считается не по количеству полностью верных ответов, а по взвешенной формуле

## Отображение в интерфейсе

В результатах тестирования для каждого вопроса отображаются:

- тип вопроса
- уровень сложности
- коэффициент ответа `r_i`
- статус зачета:
  - `Полный зачет`, если `r_i = 1`
  - `Частичный зачет`, если `0 < r_i < 1`
  - `Без зачета`, если `r_i = 0`

Итоговый процент на странице результата и в Excel-отчете рассчитывается именно по взвешенной формуле, а не как простая доля полностью верных ответов.
